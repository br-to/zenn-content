# Kalshiの意味

Kalshi は、予測市場を制度の外で拡張するんじゃなくて、制度の中で正面から成立させようとした存在です。

ここが Polymarket と決定的に違います。Polymarket がコード、トークン、透明性、グローバル性から広がったのに対して、Kalshi は規制、許認可、監督、市場運営の正統性から広がりました。つまり予測市場へのアプローチが根本的に異なるわけです。

# event contractとしての予測市場

Kalshi の文脈では、予測市場はしばしば event contract として理解されます。

これは特定の出来事が起きたかどうか、ある閾値を超えたかどうかといったイベントに連動して価値が決まる契約です。金融デリバティブの一種として扱われるので、CFTCの監督下に入ります。

# 制度がもたらすもの

規制市場型の強みは、ルールと執行の明確さです。

KYC、アクセス制御、市場監視、不公正取引への対応などが取り入れやすくなります。制度の中にいることで、ユーザーは「このプラットフォームは信頼できる」という安心感を得やすい。特に金融リテラシーが高いユーザーには、監督されている市場という点が大きなメリットになります。

# 規制型の限界

ただし、規制型だから無敵というわけじゃありません。

扱えるテーマに制約がかかりやすくて、政治、スポーツ、戦争、暗殺、地政学など社会的にセンシティブな市場では、金融商品として許容されるかがすぐに問題になります。Kalshiも政治市場をめぐって何度も揉めていますよね。

# Polymarketとの対比

Kalshi と Polymarket を比べると、両者は同じ予測市場でも土台がかなり違います。

Kalshi は制度を信頼の源泉にして、Polymarket は透明性とオープンな構造を信頼の源泉にします。どちらが優れているかという話じゃなくて、予測市場という仕組みをどう社会に位置づけるかの哲学の違いだと思います。
